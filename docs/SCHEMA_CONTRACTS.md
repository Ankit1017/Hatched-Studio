# Schema Contracts

## Location

Asset schema files live in:
- `main_app/schemas/assets/*.json`

Naming:
- `<intent>.v1.json`
- for spaced intents, underscore form is used (e.g. `data_table.v1.json`)
- scaffold script always emits underscore filenames

## Runtime Validation

Validation service:
- `main_app/services/agent_dashboard/schema_validation_service.py`

Lifecycle stage:
- `validate_schema` between `normalize_artifact` and `verify_result`

Output:
- validation summary stored in `artifact.provenance.schema_validation`

Control flag:
- `SCHEMA_VALIDATE_ENFORCE`
  - `true`: schema errors fail the tool
  - `false`: schema issues are recorded but execution continues

## Scaffold Flow

Generated by:

```bash
python scripts/scaffold_tool_plugin.py --intent "new asset" --kind structured
```

Then validate with:

```bash
python scripts/validate_plugin_specs.py
```
